### Vitest

Використовуємо для тестування **Vitest** замість Jest (основна причина - ми на лекціях вивчали Jest, зараз є можливість спробувати Vitest). Він не дуже відрізняється по принципу роботи, але на деякі моменти акцентую увагу:

_Раджу встановити окремий плагін для Vitest якщо користуєетесь VsCode (у webstorm не знаю). Він дозволяє тестувати через свою вкладку, додає до коду в тестах позначки та дає можливість увімкнути відображення покриття коду тестами (дуже зручно як на мене). Плагін так і називається Vitest_

**Тести запускаються наступним чином:**

1. Звичайний тест з виводом інформації у терміналі (як і в Jest):

   > npm run test

2. Тест який відкриває UI інтерфейс у браузері:

   > npm run test:ui

3. Тест який робить html кожного елементу з описом наскільки елемент покрит тестами (зʼявляється папка coverage, там можете подивитись). Папка додана до gitignore, тому видаляти перед пушом необовʼязково:
   > npm run test -- --coverage

У папці **feed** є тестова кнопка, яку можна протестувати, заради прикладу

# Структура

Можливо вже побачили що структура трохи не така, з якою ми працювали на лекціях, я взяв за основу принцип [Featured-Sliced Design](https://feature-sliced.design/ru/docs/get-started/overview). Для командної роботи буде трохи легше розібратись і не повинно буде бардака. Це не дуже складно, тому тапками не кидатись, зараз логіка навіть зрозуміліша і простіша.
**app** - тут зібрана логіка для рендеру нашого сайту. Роутінг, стани, глобальні стилі та інше що цього стосується. Тут ми майже нічого не створюємо (окрім вище переліченого)
**pages** - це наші готові сторінки над якими кожен окремо працює. Ваша папка згідно сторінки, працюйте як вважаєте за потрібним. На папці feed можете побачити простий приклад як рекомендовано робити, тобто додаткові папки **api** (думаю не треба говорити що тут має бути :)) та **ui** (ну тут наші компоненти які використовуються саме на цій сторінці). У самій папці (наприклад **feed**) ми вже створюємо нашу сторінку і збираємо все до купи (FeedPage.jsx), прокладуємо роутінг (якщо не одразу це зробили) і в нас готова сторінка. Звісно це рекомендація, ви можете робити як хочете, але на мою думку це набагато зручніше, ніж коли всі компоненти зі всіх сторінок будуть в одному місці, а сторінки десь в іншому. Головне не створюйте що стосується вашої сторінки поза межами своєї папка :)
**shared** - папка з reusable components та логікою, тобто компоненти або запити які будут на всіх або багатьох сторінках. Типу кнопок, хедер, інпути і таке інше, думаю зрозуміло.

**test** - папка для конфігурації тестів, не чіпаємо
